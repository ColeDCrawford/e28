DIY SPA

Do It Yourself Single-Page Application

Ultimately, we’ll transform our ZipFoods application into a true SPA using the vue-router library.

Before we do that, though, let’s explore how we could start to build a SPA using the techniques we already have. This will be an opportunity to think critically about how SPAs work, and also practice existing skills.

Expectations of a SPA

  • Visitor can click hyperlinks that lead to different “pages” in the application.
  • The actual web page is never refreshed; a new request is never sent to the server. All page switching is simulated with JavaScript.

Our “pages”

To start, we'll have 3 “pages”, each made via their own component:

Page 1) Home page

For the home page, create a new /components/pages/HomePage.vue component that:

  • displays the ZipFoods tagline that was previously in App.vue
  • Displays the featured categories (via the ShowFeatured component)

Page 2) Products page

For the products page, repurpose the existing ShowProducts component, renaming/relocating it to /components/pages/ProductsPage.vue.

Page 3) Categories page

For the categories page, create a new /components/pages/CategoriesPage.vue component that displays a list of unique categories.

Here’s a template you can use:

<template>
    <div>
        <h2>Categories</h2>
        <ul class='cleanList'>
            <li v-for='(category, id) in categories' :key='id'>{{ category }}</li>
        </ul>
    </div>
</template>

And here’s a computed property you can use to filter unique categories out of the products data:

categories() {
    let categories = this.products.map(product => product.categories);
    let mergedCategories = [].concat.apply([], categories);

    // Return unique, sorted categories
    return [...new Set(mergedCategories)].sort();
}

Test the pages

After completing the above steps, you could test that all three ”page” components are working my making them all display in the root App component:

<home-page></home-page>
<categories-page></categories-page>
<products-page></products-page>

Content swapping - Approach A

To accomplish the multi-page application effect, we need to swap out the content on the page, giving the impression that the page is switching.

One way we could accomplish this is by introducing a page data property that is changed from navigation links and used determine which page component to show.

Example:

<!-- Added to the App.vue template -->
<nav>
    <ul>
        <li>
            <a href='#' @click='page = "home"'>Home</a>
        </li>
        <li>
            <a href='#' @click='page = "products"'>Products</a>
        </li>
        <li>
            <a href='#' @click='page = "categories"'>Categories</a>
        </li>
    </ul>
</nav>

<home-page v-if='page == "home"'></home-page>
<products-page v-if='page == "products"'></products-page>
<categories-page v-else-if='page == "categories"'></categories-page>

The above code would require the following data properties:

page: 'home', /* Default page to start with */

Content swapping - Approach B

Let’s refactor the above code so that each link in our navigation is generated from a list of pages. This will remove some redundant code and make it easy to add new links in the future.

We’ll also take advantage of a feature called dynamic components that lets you dynamically display a component like this:

<component v-bind:is='my-component'></component>

Here’s the refactored code:

<nav>
    <ul>
        <li v-for='link in links' :key='link'>
            <a href='#' @click='page = link'>{{ link }}</a>
        </li>
    </ul>
</nav>

<component v-bind:is='linkComponents[page]'></component>

Data properties:

/* Default page to start with */
page: 'home',

/* Store links in an array to maintain order */
links: ['home', 'products', 'categories'],

/* Map links to the appropriate component */
linkComponents: {
    home: 'home-page',
    products: 'products-page',
    categories: 'categories-page'
}

What we’re missing: Dynamic URLs

Our DIY approach to making a SPA is off to a good start - but we’re missing a key feature:

When users visit different pages on a site, they expect the URL to update and reflect the current page they’re on.

Not only is this important to help visitors identify where they are on a site, it’s necessary for our visitors to be able to bookmark and copy/paste/share links to our site.

Rather than continuing down the DIY path, let’s switch gears to working with vue-router, which will solve the problem of dynamic URLS - as well as many other SPA features we’ll want to address.
