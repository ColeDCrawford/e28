Vue.js Introduction

Vue is a progressive framework for building user interfaces. Unlike other monolithic frameworks, Vue is designed from the ground up to be incrementally adoptable. The core library is focused on the view layer only, and is easy to pick up and integrate with other libraries or existing projects. On the other hand, Vue is also perfectly capable of powering sophisticated Single-Page Applications when used in combination with modern tooling and supporting libraries. -ref

Versions

  • Vue.js is currently at version 2, but v3 is close to being released.
  • v3 adds new features, some new ways of doing things, and an optimized infrastructure for how Vue works behind the scenes.
  • The v2 way of doing things will continue to work in v3; v3 is “additive”.
  • Since v3 is not yet released, we’ll start this semester with v2 and discuss v3 differences at some point in the semester. Reasoning:
    • Even when v3 is released, it will take some time for Vue related tools and dependencies to update to work with v3.
    • v2 code in existing projects, tutorials, etc. will continue to be around for a while, so it’ll be useful to understand how it works.

Documentation

  • https://vuejs.org
  • https://vuejs.org/v2/guide
  • https://vuejs.org/v2/api
FYI

Vue docs for v3 beta can be viewed at https://v3.vuejs.org

Importing the framework

In the head of your index.html page:

<script src='https://cdn.jsdelivr.net/npm/vue/dist/vue.js' defer></script>
  • We’re using the development version - full warnings, debug mode, non-minified
  • Loading from an external source; CDN - Content Delivery Network
  • Later in the semester, we’ll look at using NPM (Node Package Management) to import, and we’ll address using the production version (no warnings, minified)
  • Including the vue.js script will register the global variable Vue

New Vue instance

In script.js, create a new Vue instance using the Vue function; this function will receive an object of options.

Options are how we’ll configure the properties and behaviors of our application.

// The Vue instance
const app = new Vue({
  // Options
})

el Option

First option: el

Specifies what DOM element in our HTML that Vue will “mount” on.

To use it, we first need a DOM element to mount to, so add the following div to the body of your page: 

<div id='app'></div>

Then add the el option and configure it to #app (the id of the div):

const app = new Vue({
    el: '#app'
});

Test it

  • Note the Vue in development mode message in the console
  • Note the Vue tab in your web inspector is active (assuming you’ve installed Vue dev tools)
  • Note how you can access the Vue instance from the console
    • Accessible via the app variable you created
    • Instance properties are prefixed with a dollar sign, e.g. app.$el

data Option

Next option: data

The data option lets us pass an object of user-defined data to the Vue instance, e.g.:

var app = new Vue({
    el: '#app',
    data: {
        playerName: 'Susan'
    }
});

Text interpolation

Now that we have some data, we want to start to bind it to the page.

One way to due this is via text interpolation using the double curly bracket aka “mustache syntax”:

<div id='app'>
    <p>Player: {{ playerName }}</p>
</div>

Reactive data

Data in our Vue app is all reactive - meaning if it changes in one place, that change will propagate wherever else that data is being used.

To demonstrate this, we can update playerName via the console and see the change reflected where it’s printed on the page:

app.$data.playerName = 'Joe'
FYI

All user-defined data is aliased, so you can also use the shorthand app.playerName when accessing it via the console.

Directives / v-model

So far, our player’s name was hardcoded. It would be more useful if the visitor could type in their name.

To accomplish this, we’ll look at another approach to binding, this time using something called v-model.

As an example, add an HTML text input with the attribute v-model='playerName':

<div id='app'>
    <label>What’s your name? <input type='text' v-model='playerName'></label>
    <p>Player: {{ playerName }}</p>
</div>

v-model is our first example of a Vue directive.

Vue directives...

  • are attributes you add to your HTML elements
  • are always prefixed with v-
  • provide a way to reactively apply side effects to the DOM

The v-model directive...

  • creates a two-way binding between a data property and a form input
  • two-way binding:
    • if the data property playerName changes, so will the value of the text input
    • if the value of the text input changes, so will the playerName data property

To observe the reactive binding, try any of the following:

  • Type something into the text input
  • Alter app.playerName via the console
  • Alter app.playerName via Vue dev tools

Conditional rendering with v-if

Another useful directive is v-if which allows you to conditionally toggle the presence of an element on the page.

The v-if directive accepts a boolean expression; if that expression returns true, the corresponding element will be visible.

As an example, lets make it so that the player name output only displays once we have a player name:

<div v-if='playerName != null'>
    Player: {{ playerName }}
</div>

In order for the above example to work, update your playerName data property so it’s null on page load:

const app = new Vue({
    el: '#app',
    data: {
        playerName: null
    }
});

Other conditional-related directives include v-else, v-else-if, and v-show, which we’ll dig into more detail later.

Directive arguments / Event listening with v-on

Some directives can take arguments which are added following a colon after the directive name.

Here’s an example with the v-on directive, which is accepting the argument click: 

<button v-on:click='doSomething'>Do Something</button>

Other acceptable arguments for v-on include any native browser event as: click, change, mouseover, mouseout, keydown, etc.

For an example, imagine our game needed the user to make guesses, and we want to keep track of those guesses.

To accomplish this, we’ll add two data properties: guess and guesses...

const app = new Vue({
    el: '#app',
    data: {
        'playerName': null,
        guess: null, // New
        'guesses': [] // New
    }
});

...and set up a text input to receive a guess, plus a button to record the guess:

<label>Guess the number <input type='text' v-model='guess'></label>
<button v-on:click='guesses.push(guess)'>Guess</button>

We can also use a v-for directive to loop through and output each guess:

<ul>
    <li v-for='thisGuess in guesses'>{{ thisGuess }}</li>
</ul>

Cloaking

You may observe that when your page is first loaded, any text interpolations show up in their raw form. For example, if we had this HTML:

<div id='app'>
    {{ playerName }}
</div>

You’d see the following on initial page load, very briefly:

Once the JavaScript is loaded and the Vue instance is created and bound to the DOM, though, the content of the page is rendered as expected.

That momentary “blip” of un-rendered content, though, looks odd to the user.

To avoid it, Vue uses a “cloaking” technique to hide the app before it’s ready.

To do this, add the v-cloak directive to your app:

<div id='app' v-cloak>
    [...]
</div>

Then, style this directive in your CSS so it’s invisible:

[v-cloak] {
    display: none
}

Once the Vue instance is compiled, the v-cloak directive will be removed, so the app will become visible.

Summary

  • When we initialize a Vue instance it’s mounted to an element on the page
  • The Vue instance is configured via options such as el and data
  • One way data can be bound to the page is by using text interpolation using the mustache syntax {{ }}
  • Vue directives appear in the form of a prefixed HTML attribute (v-); they’re used to apply side effects to elements.
  • Example directives include: v-model, v-if, v-on, v-for
  • Some Vue directives accept arguments (e.g. v-on:click)
  • The directive v-cloak can be used to hide unrendered Vue content seen on initial page load.
