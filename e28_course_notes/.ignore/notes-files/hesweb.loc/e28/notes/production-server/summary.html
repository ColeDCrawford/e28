Summary

By the time you get to this note set, you will have accomplished a lot, including several “big picture” setup steps:

  • Downloading and setting up your XAMPP local server
  • Installing/configuring git
  • Setting up a SSH connection between your computer and Github.com
  • Creating a new git repository (e28) on Github.com and cloning it to your computer
  • Creating a new DigitalOcean server with a SSH key connection to your computer, as well as a new key pair to connect to Github.com
  • Cloning your e28 repository from Github.com to your production server
  • Configuring a real-world domain to access your production server

Assuming nothing goes awry - these latter steps should not have to be completed again this semester.

The steps you will reproduce, though, are the ones necessary to set up a new application. To help with this, below is a summary of those steps.

New app summary

Step 1. Create the application directory

Locally, within your e28 repository, create a new directory that will contain the new application.

Add a README.md and index file to this directory.

Step 2a. Local domain - VirtualHost

Add a new VirtualHost block to your local server’s Apache config file.

Mac:

$ nano /Applications/XAMPP/etc/extra/httpd-vhosts.conf

Windows:

$ nano c:\xampp\apache\conf\extra\httpd-vhosts.conf

Example VirtualHost template:

<VirtualHost *:80>
    ServerName appname.loc
    DocumentRoot /path/to/htdocs/e28/appname
    <Directory /path/tohtdocs/e28/appname>
        AllowOverride All
        Options -Indexes
        Require all granted
    </Directory>
</VirtualHost>

Stop and restart your local server for these changes to take effect.

Step 2b. Local domain - hosts routing

Edit your computer’s hosts file to register this new domain.

Mac:

sudo nano /private/etc/hosts

Windows:

elevate nano c:\Windows\System32\drivers\etc\hosts

Example line to add:

127.0.0.1 appname.loc

Step 3. Synchronize with Github.com

Locally - Stage, commit, and push the files for your new application to your e28 repository on Github.com.

From within your /htdocs/e28 directory:

$ git add --all
$ git commit -m "New application"
$ git push

Step 4. Synchronize with your DigitalOcean server

SSH into your DigitalOcean droplet:

$ ssh root@your.do.ip.address

Navigate to your e28 repository:

$ cd /var/www/html/e28

Pull the latest changes from Github.com:

$ git pull

Step 5. Production domain - VirtualHost

On your production server, add a new VirtualHost block to your Apache config file:

$ nano /etc/apache2/sites-enabled/000-default.conf

Example VirtualHost template:

<VirtualHost *:80>
    ServerName appname.yourdomain.com
    DocumentRoot "/var/www/html/e2/appname"
    <Directory "/var/www/html/e2/appname">
        AllowOverride All
        Options -Indexes
        Require all granted
    </Directory>
</VirtualHost>

Save your changes and restart Apache:

$ service apache2 restart

Development workflow

Once an application is set up, your development workflow can be summarized as follows:

  • Develop
    • Make changes to your application's codebase locally
    • Test/tweak those changes until you're happy with them
  • Track
    • Stage all the files you wish to "go live" with (git add)
    • Create a commit of all staged files (git commit -m "Commit message goes here")
    • Push your commit to Github.com (git push)
  • Deploy
    • SSH into your DigitalOcean server
    • Navigate to your e28 repository (e.g. cd /var/www/html/e28)
    • Pull recent commits from Github.com (git pull)
    • View app in browser via production URL, confirming everything is working as expected
