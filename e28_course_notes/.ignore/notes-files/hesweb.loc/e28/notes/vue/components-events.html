Component Events

Properties provide a way to pass information from parent Vue instances to child Vue instances, but how can we go the other way, passing data “up” to the parent?

Answer: Events

Example: In an earlier iteration of the round-detail component, we had set up a deleteRound method that caused the component to get hidden when the method was invoked.

Now that our round data is programmed into the root Vue instance, though, we need to improve on this so when the Delete round button is clicked within the child component, it actually updates the data in the parent root Vue instance.

To do this, first add a new deleteRound method to the root Vue instance as shown in this example:

// root Vue instance
let app = new Vue({
    el: '#app',
    data: {
        rounds: [
            { number: 1, winner: 'Player' },
            { number: 2, winner: 'Player' },
            { number: 3, winner: 'Computer' },
        ]
    },
    methods: {
        // NEW METHOD
        deleteRound(number) {
            this.rounds = this.rounds.filter((round) => round.number != number);
        }
    }
});

Next, when invoking the round-detail component, give it a v-on event listener.

Instead of using a traditional event such as click, though, we’ll define a custom event called delete-round which will invoke the deleteRound method we just created. When invoking deleteRound, we’ll pass a special parameter $event which will contain the data passed “up” from the child component. 

<div id='app' v-cloak>
    <round-detail v-for='round in rounds' 
        v-on:delete-round='deleteRound($event)' 
        v-bind:key='round.number'
        v-bind:number='round.number' 
        v-bind:winner='round.winner'>
    </round-detail>
</div>

Finally, in our component template, we’ll set up a click listener that uses Vue’s built-in instance method $emit to emit an event called delete-round, passing with it the number: 

<script type='text/x-template' id='round-detail'>
    <div v-if='!deleted' class='results'>
        <ul>
            <li>Round #: {{ number }}</li>
            <li>Winner: {{ winner }}</li>
        </ul>
        <button v-on:click="$emit('delete-round', number)">Delete round</button>
    </div>
</script>

Summary:

  • Child instances can $emit custom events, which can include data
  • Parent instances can listen for these custom events and receive the data ($event)

Event case conventions

Unlike component names and prop names, event names won’t be automatically translated between camelCase/PascalCase and kebab-case.

This is because event names are never actually used within the JavaScript code, only the HTML code. And because of this, they should always be defined using kebab-style casing.

Data flow

There’s a Vue instance property called $root which provides access to a child’s parent instance.

Knowing this, you could technically alter the root Vue instance’s rounds data property from directly within the child component, instead of triggering the root Vue instance’s deleteRound method to handle the task: 

Vue.component('round-detail', {
    // [...]
    methods: {
        deleteRound(number) {
            console.log('Deleted round ' + number + ' from the component.');
            function isMatchingRound(round) {
                return round.number != this;
            }
            this.$root.rounds = this.$root.rounds.filter(isMatchingRound, number);
        }
    },
    // [...]
});

However, this is not considered a best practice- child components should not mutate a parent’s state as it can make the app’s data flow harder to understand.
