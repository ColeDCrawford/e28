Single file components (SFC)

Summary

Single file components (SFC) allow you to encapsulate all the code for a Vue component into one file.

This includes the template, JS, and CSS.

The advantage of SFCs is they can then easily be imported and re-used throughout your project.

SFCs require a build system to compile them into browser-ready JavaScript.

Details covered here: Build systems introduction...

Creating SFCs

Single file component files end with a .vue extension and the file name matches the component name (e.g. RoundDetail).

Every SFC is broken down into 3 parts:

<template>
</template>

<script>
export default {
    // ... component options    
};
</script>

<style>
</style>

On initial introduction, this mixing of HTML, JavaScript, and CSS together in one file might feel “wrong” and contradictory to everything we’re taught about separation of concerns (SoC) when learning about these technologies.

However, the designers of Vue argues that we‘re not loosing SoC - we’re just doing it in a different way. Rather than separating concerns based on technologies, we’re separating based on modules/entities within our application. Go here to read more about this approach...

All that being said - if you’re not on board with this different way of looking at SoC, note that scripts and styles can be included externally if you prefer, e.g.:

<script src="/src/js/round-detail.js"></script>
<style src="/src/css/round-detail.css"></style>

Using SFCs

To use a SFC you import it as a module:

import RoundDetail from '/@components/RoundDetail.vue'

And then you list it in the components option of whatever Vue instance you wish to use the component:

components: {
    'round-detail': RoundDetail
},

Vetur - Vue tooling for VS Code

To get correct syntax highlighting in your .vue SFC files install the VSCode extension Vetur.

  • https://vuejs.github.io/vetur
  • https://marketplace.visualstudio.com/items?itemName=octref.vetur

Scoped CSS

When setting up the CSS in your SFC, you have the option of including/excluding the scoped attribute:

<style scoped></style>

When scoped, the CSS will only apply to elements of that component.

For example, let’s say you set a style for h2 elements:

<style scoped>
    h2 { 
        color:red 
    }
</style>

In traditional CSS, this would apply the style to all h2 elements on the page. But with scoped CSS, it will only apply the style to h2 elements within the component’s template.

Mixing scoped/non-scoped styles

You can include both scoped and non-scoped styles in the same component:

<style>
/* global styles */
</style>

<style scoped>
/* local styles */
</style>

How it works

Behind the scenes, Vue accomplishes scoped CSS by generating unique attributes to be used as selectors that are applied to your component’s elements.

Example of how scoped CSS is rendered:

<style>
.example[data-v-f3f3eg9] {
  color: red;
}
</style>

<div class='example' data-v-f3f3eg9>Hello World</div>

CSS Pre-processors

You can use pre-processors CSS languages like SCSS in your components by setting the lang attribute:

<style lang='scss'>
$green: #77ff33;

.winner {
    background-color: $green;
}
</style>

This will work ”out of the box” if using the integrated setup with Laravel Mix.

If using the comprehensive setup with Vue CLI, you’ll have to install the loader for your CSS pre-processor, e.g.:

$ npm install -D sass-loader sass

Learn more...

Visit https://vue-loader.vuejs.org/ to learn more about the capabilities of SFCs using vue-loader.
