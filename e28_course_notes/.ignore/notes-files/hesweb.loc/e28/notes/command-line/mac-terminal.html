Mac - Terminal

On Macs, Command Line is accessed via an included application called Terminal.

Find it in /Users/ComputerName/Applications/Utilities/ or just type Terminal into Spotlight.

If you’re new to command line, the first thing you should do after loading Terminal is run through the command line primer.

Beyond that, the following details explain how to setup aliases, which will help speed up your day-to-day workflow.

Aliases

Aliases provide shortcuts to commonly used commands.

For example, in lecture, you’ll see me frequently use the command ll which is actually an alias for ls -laFG (list command with some commonly used flags: include hidden files, file types, color coded output, etc.).

Another alias you may see me use is htdocs which is an alias to take me to my local server’s htdocs directory (cd /Applications/XAMPP/xamppfiles/htdocs).

Setting aliases

One way to set up aliases in Terminal is to open the shell config file ~/.bash_profile (or .zprofile if you’re using Catalina/zsh).

Within this file, you can define aliases following the examples shown below. Feel free to customize/add your own aliases as needed.

Important: After editing ~/.bash_profile you need to restart terminal or run source ~/.bash_profile to make the changes take effect.

# ------------------------------------
# ALIASES
# ------------------------------------

# Ideal directory listing
alias ll="ls -laFG"
# -l list in long format
# -F Display a...
#       slash (`/') immediately after each pathname that is a directory,
#       asterisk (`*') after each that is executable,
#       at sign (`@') after each symbolic link
#       equals sign (`=') after each socket,
#       percent sign (`%') after each whiteout,
#       vertical bar (`|') after each that is a FIFO.
# -G = Color
# -a = Show hidden files

# Ask before removing files
alias rm='rm -i'

# Example alias for SSH'ing into a server
alias myserver="ssh user@111.111.111.111"

# Example alias for quickly getting to a commonly used directory
alias htdocs='cd /Applications/XAMPP/xamppfiles/htdocs'

# Open your shell config file in text edit (switch to ~/.zprofile if you're using zsh)
alias bashedit='sudo open -a TextEdit ~/.bash_profile'

# Force terminal to recognize changes to .bash_profile (switch to ~/.zprofile if you're using zsh)
alias bashrefresh='source ~/.bash_profile'

# Edit XAMPP Apache Virtual Hosts config file
alias vhosts='sudo open -a TextEdit /Applications/XAMPP/etc/extra/httpd-vhosts.conf'

# Edit XAMPP Apache config file
alias vhosts='sudo open -a TextEdit /Applications/XAMPP/xamppfiles/etc/httpd.conf'
