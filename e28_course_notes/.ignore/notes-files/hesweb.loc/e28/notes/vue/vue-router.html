What is routing?

In a very basic web site, there’s a correlation between a URL and directories/files on a server.

For example, if we visit a URL like http://example.com/apply/project-manager the path /apply/project-manager would imply there’s a file being loaded from /apply/project-manager/index.html.

Robust web applications - regardless of whether they’re server or client-side focussed - rarely rely on this model anymore.

Instead, they use a routing system, where paths on the URL do not correlate to directories/files; instead they correlate to instructions or actions that the application responds to.

So a URL http://example.com/apply/project-manager could, generally speaking, be translated to a set of instructions/procedures such as Show job listing for project manager.

In a routing system, all traffic is typically processed through an entry-point file that reads the given request, dissects the path, and invokes the application to return the appropriate response based on the requested path.

In our client-side focussed applications, that entry-point file is index.html.

In order for this to work, servers must be configured to override their default behavior of mapping paths to directories/files. Instead, the server is configured to treat every request as a request to index.html.

Install Vue Router

Now that we understand the general idea of how a routing system works, let’s get one set up in our ZipFoods application.

To do this, we’ll utilize Vue Router, the official routing library for Vue.js, which offers the following features:

  • Dynamic URLs (HTML5 history mode or hash mode, with auto-fallback in IE9)
  • Modular, component-based router configuration
  • Route params, query, wildcards
  • View transition effects powered by Vue.js' transition system
  • Links with automatic active CSS classes
  • Nested route/view mapping
  • Customizable scroll behavior

Begin by installing vue-router in the zipfoods directory:

$ npm install vue-router

This will download vue-router to your node_modules directory and update your package.json file, listing vue-router as a new dependency.

Set up Vue Router

In /src/main.js import and register the VueRouter:

// src/main.js
import VueRouter from 'vue-router';
Vue.use(VueRouter);

Next, create a new VueRouter instance, passing in an option routes that is set to an array of routes, each containing a path and a corresponding component to load when that path is triggered. Additionally, add an option mode that is set to history. 

// src/main.js
const router = new VueRouter({
    routes: [
        { path: '/', component: HomePage },
        { path: '/products', component: ProductsPage },
        { path: '/categories', component: CategoriesPage },
    ],
    mode: 'history',
})

You’ll now need to import your components into main.js:

// src/main.js
import HomePage from '@/components/pages/HomePage.vue';
import ProductsPage from '@/components/pages/ProductsPage.vue';
import CategoriesPage from '@/components/pages/CategoriesPage.vue';

Finally, in the code that creates your root Vue instance, add a router option that is set to the router instance you just defined:

// src/main.js
new Vue({
    router: router, // <-- NEW
    render: h => h(App),
}).$mount('#app')
FYI

The process for setting up Vue router in Vue 3 is a little different; see Vue 3: Vue router for more details

Utilize Vue Router

With our router set up in main.js, we next want to modify our App component (/src/components/App.vue) to utilize the routing system.

To do this, first swap out your old nav from our “DIY SPA” examples with this new nav that uses Vue Router's <router-link> component to generate each link:

<nav>
    <ul>
        <li>
            <router-link
                v-for="link in links"
                v-bind:key="link"
                v-bind:to="paths[link]"
                exact
                >{{ link }}</router-link
            >
        </li>
    </ul>
</nav>

Then, below this nav and in place of our dynamic component from our “DIY SPA” examples, we’ll add Vue Router’s <router-view> component, which is where Vue router will inject our components when they’re requested:

<router-view></router-view>

Finally, we’ll set up the following data properties which are used to define our links in the template: 

/* Store links in an array to maintain order */
links: ['home', 'products', 'categories'],

/* Map links to the appropriate component */
paths: {
    home: '/',
    products: '/products',
    categories: '/categories',
},

Test it...

With the above code in place, access your site and click each link in the navigation. As you do this, observe the following:

  1. Each link triggers the loading of a new component

  2. There is no page refresh happening as links are clicked

  3. The “active” link has a class router-link-active automatically applied to it (this class is pre-styled in the provided zipfoods.scss) ] In lecture I will demonstrate/explain what happens when we experiment with the following changes:

  4. Using a regular hyperlink instead of <router-link>

  5. Removing the exact attribute from the <router-link>

  6. Removing the mode : 'history' option from our Vue router configuration

Dynamic route matching

All of the paths we’ve defined so far have been very specific, e.g. /products, /categories.

Often times, though, it’s useful to have paths that can accept dynamic segments, allowing us to pass information as part of the path.

As an example, let’s build a “page” to show the details for an individual product that uses a dynamic path segment to indicate which product (by id) should be known.

In other words, we want to be able to invoke /products/x where x is the id of the product we wish to load.

To begin, define a new route which will invoke a ProductPage component:

const routes = [
    // [...]
    { path: '/products/:id', component: ProductPage, props: true },
]

Note the following two things about this route that make it unique from the previous routes we set up:

  1. The use of :id in the path; the : indicates it’s a dynamic segment, and it could match any value.
  2. The addition of props: true which will make it so the values from our dynamic segments will be available as props within our component.

To accommodate this route, create a new component, /src/components/pages/ProductPage.vue: 

<template>
    <div id='product-page'>
        <h1>Product {{ id }}</h1>
    </div>
</template>

<script>
export default {
    props: ['id'], // comes from our dynamic segment
    data() {
        return {
        };
    }
};
</script>
