Integrated build - Local

For our first example, we’ll look a integrating a Vue.js build system into an existing project. As our “existing project” we‘ll use the app we created last lecture (week6) as an example.

Goals:

  • Use NPM to manage our external dependencies (e.g. Vue)
  • Ability to use SFC .vue files
  • Nicely bundled JS files
  • Hot module reloading

To get started, make a copy of your week6 directory and name it week8 - we’ll use this as our starting point.

NPM setup

To get NPM set up in this project, create a NPM config file called package.json in the root of your project with these contents:

{
    "private": true,
    "scripts": {
        "dev": "cross-env NODE_ENV=development node_modules/webpack/bin/webpack.js --progress --hide-modules --config=node_modules/laravel-mix/setup/webpack.config.js",
        "watch": "npm run dev -- --watch",
        "hot": "cross-env NODE_ENV=development node_modules/webpack-dev-server/bin/webpack-dev-server.js --inline --hot --config=node_modules/laravel-mix/setup/webpack.config.js",
        "prod": "cross-env NODE_ENV=production node_modules/webpack/bin/webpack.js --no-progress --hide-modules --config=node_modules/laravel-mix/setup/webpack.config.js"
    },
    "devDependencies": {
        "cross-env": "^7.0",
        "laravel-mix": "^5.0.1",
        "vue-template-compiler": "^2.6.12"
    },
    "dependencies": {
        "vue": "^2.6.12"
    }
}

Here’s a breakdown of each section of the above config:

  • private: Indicates we don’t intend to publish this repository on the NPM registry.
  • scripts: A series of shortcuts for commands we’ll run to build our project. More details on each command below.
  • devDependencies: Packages we only need locally in our development environment. All the packages listed in the provided template are going to be used as part of our build process.
  • dependencies: Packages we need on both our development and production environment.

After you’ve added the above config file, run the following command:

$ npm install

NPM will process your package.json file and download all your dependencies into a directory called node_modules. Additionally, it will record the specific version numbers that were downloaded in package-lock.json.

Laravel Mix

Now that we have the Laravel Mix dependency installed, we can use it to process our JavaScript files using Webpack.

“Laravel Mix is a clean layer on top of webpack to make the 80% use case laughably simple to execute. Most would agree that, though incredibly powerful, webpack ships with a steep learning curve. But what if you didn’t have to worry about that?” https://laravel-mix.com/docs/5.0/basic-example

To configure Laravel Mix, create a new file webpack.mix.js in the root of your project with the following code:

let mix = require('laravel-mix');

// https://laravel-mix.com/docs/mixjs
// source: src/script.js
// output: js/master.js
mix.js('src/script.js', 'js/master.js');

That’s it! With just a handful of lines of code, Mix will take care of the following:

  • Compiling modern ES6 features into cross-browser friendly JS
  • Building and compiling Vue Single File Components
  • Setting up hot module replacement

Adapt code

We now have the software and configurations needed for our build system. Our next step is to adapt our existing project code so that we can harness this build system. Here are the steps to do that...

Step 1. Reorganize

Within week8/, create a new directory called src. This is where we’ll put all our pre-compiled files.

Move your scripts.js file into this directory.

Also within src, create a components/ directory where you can put the RoundDetail.vue file we’ve already created.

End result:

week8/
    src/
        components/
            RoundDetail.vue
        scripts.js
    index.html

Step 2. Update script.js

Update script.js to...

  • import Vue module
  • import RoundDetail module
  • add a components option to root Vue instance
import Vue from 'vue';
import RoundDetail from './components/RoundDetail.vue'

let app = new Vue({
    el: '#app',
    // NEW: Add a components option
    components: {
        'round-detail' : RoundDetail
    },
    // [...etc...]

Step 3. Update index.html

Remove reference to Vue.js from the CDN:

<script src='https://cdn.jsdelivr.net/npm/vue/dist/vue.js' defer></script>

Remove reference to our source JS file:

<script src='script.js' defer></script>

Add reference to our compiled/master JS file:

<script src='/js/master.js' defer></script>

Run the build system

With everything set up, we’re now going to run our build system.

We’ll do this by first invoking our dev script from package.json via this command:

$ npm run dev

This will invoke webpack using our laravel-mix config which will compile our src/script.js file to js/master.js

We could run this same command every time we make changes to our JavaScript files, but that would quickly become tedious. Instead, we can use the watch script which will watch for changes in our JavaScript files and re-run the build system whenever it detects changes:

$ npm run watch

Finally, if we want to take advantage of hot module reloading, we’ll want to run our site using a webpack dev server, and that can be triggered via the hot script:

$ npm run hot

Running this script will produce a localhost address you can use to access your project (e.g. http://localhost:8080/), and whenever you make changes to the JavaScript in your project, it will automatically get updated in the browser.

“Hot Module Replacement (or Hot Reloading) allows you to, not just refresh the page when a piece of JavaScript is changed, but it will also maintain the current state of the component in the browser. As an example, consider a simple counter component. When you press a button, the count goes up. Imagine that you click this button a number of times, and then update the component file. Once you do, the webpage will refresh to reflect your change, but the count will remain the same. It won’t reset. This is the beauty of hot reloading!” https://laravel-mix.com/docs/hot-module-replacement

FYI

Using the webpack dev server supersedes the need to use XAMPP/Apache in local development.
