Component Slots - content distribution

In addition to props, another way to pass information to components is via slots.

The things you can accomplish with props and slots are similar, they just have different approaches.

Summary of how a basic slot works:

  • Put a <slot></slot> element in your component’s template where you want content to be injected.
  • When using the component, its inner HTML content will be injected into the slot.

As an example, let’s output some extra details in our round-detail component including the result of a coin flip (e.g. heads or tails).

To do this, first update your rounds data to include a coin property. While you’re there, also set a choice property, as we’ll use that later in this example as well. 

data: {
    rounds: [
        { number: 1, winner: 'Player', coin: 'heads', choice: 'heads' },
        { number: 2, winner: 'Player', coin: 'tails', choice: 'tails' },
        { number: 3, winner: 'Computer', coin: 'heads', choice: 'tails' },
    ]
},

Then, add a new list item to your component template where the content of the item is set to <slot></slot>. The <slot></slot> element acts as an injection point for content. 

<script type='text/x-template' id='round-detail'>
    <div v-if='!deleted'>
        <ul>
            <li>Round #: {{ number }}</li>
            <li>Winner: {{ winner }}</li>
            <li><slot></slot></li>
        </ul>
        <button @click="$emit('delete-round', number)">Delete round</button>
    </div>
</script>

Finally, update the inner contents of the <round-detail> element with the content we want to inject in the slot:

<round-detail v-for='round in rounds'
    v-on:delete-round='deleteRound($event)'
    v-bind:key='round.number'
    v-bind:number='round.number'
    v-bind:winner='round.winner'>

    The coin landed on {{ round.coin }}

</round-detail>

Multiple slots via named slots

If we wanted to also report on what side (heads or tails) the player chose for each round, we could add another slot, but it’d have to be a named slot so we could distinguish from the existing slot.

Doing this requires two changes:

  • Apply a name attribute to your slots, e.g. <slot name='abc'></slot>
  • Wrap the slot content within a template element with a v-slot directive, e.g. <template v-slot:abc>Lorem ipsum dolor sit amet</template>

For example...

Template:

<script type='text/x-template' id='round-detail'>
    <div v-if='!deleted'>
        <ul>
            <li>Round #: {{ number }}</li>
            <li>Winner: {{ winner }}</li>
            <li><slot name='coin'></slot></li>
            <li><slot name='choice'></slot></li>
        </ul>
        <button @click="$emit('delete-round', number)">Delete round</button>
    </div>
</script>

Usage:

<round-detail v-for='round in rounds'
    v-on:delete-round='deleteRound($event)'
    v-bind:key='round.number'
    v-bind:round-number='round.number'
    v-bind:winner='round.winner'>

    <template v-slot:coin>The coin landed on {{ round.coin }}</template>

    <template v-slot:choice>You chose {{ round.choice }}</template>

</round-detail>

The results should look something like this:

FYI: As a shortcut to v-slot: you can use the # sign, e.g.:

<template #coin>The coin landed on {{ round.coin }}</template>

<template #choice>You chose {{ round.choice }}</template>

Slots vs. props

In the above example, and in general - whenever injecting pieces of data - I prefer the props approach as it’s less verbose.

Where I find slots to be a more natural solution is when you’re not just injecting pieces of data, but longer form content.

For a good example of this, study the base-layout component example described in the Vue.js official documentation.
