Form input binding

In the introduction we saw how the v-model directive can be used to create a two way binding between a data property and a form input.

<label>What’s your name? <input type='text' v-model='playerName'></label>
<p>Player: {{ playerName }}</p>

In this example, we worked with a basic text input.

In this note set, we’ll dig deeper to learn about how v-model works on other input types.

FYI

As you go through this information, observe how Vue facilitates a data-focussed and declarative code style. When binding values to different input types (radio, checkbox, select, etc.), Vue allows us to focus on the data while it handles the unique mechanics of working with each different input type.

Checkboxes

Toggling the true/falseness of a v-model attribute on a checkbox will:

  • Trigger a change event
  • Toggle the checked property

Example:

data: {
    showHint: true,
}

<input type='checkbox' id='showHint' v-model='showHint'>
<label for='showHint'>Show hint?</label>

<div v-if='showHint'>
    Lorem ipsum dolor sit amet, consectetur adipiscing elit.
</div>

Group of checkboxes

You can also influence a group of related checkboxes using an array of values.

For example, given the following checkboxes:

<input type='checkbox' id='faceCards' value='faceCards' v-model='config'>
<label for='faceCards'>Include face cards</label>

<input type='checkbox' id='showComputersCards' value='showComputersCards' v-model='config'>
<label for='showComputersCards'>Show computer’s cards</label>

<input type='checkbox' id='aceLow' value='aceLow' v-model='config'>
<label for='aceLow'>Ace low</label>

If the data for the config data property was ['faceCards', 'aceLow'], then just those two checkboxes would be selected.

data: {
    config: ['faceCards', 'aceLow']
}

Radios

Because radio buttons are intended to be mutually exclusive, you can bind to a property that has a single value that matches the value attribute of the radio option you wish to be chosen.

Example:

data: {
    level: 'easy'
}

Difficulty level:

<input type='radio' id='easy' value='easy' v-model='level'>
<label for='easy'>Easy</label>

<input type='radio' id='moderate' value='moderate' v-model='level'>
<label for='moderate'>Moderate</label>

<input type='radio' id='difficult' value='difficult' v-model='level'>
<label for='difficult'>Difficult</label>

Selects

Binding a property to a select will cause its option with the matching value to be selected.

Example:

data: {
    level: 'easy'
}

<select v-model='level'>
    <option value='easy'>Easy</option>
    <option value='moderate'>Moderate</option>
    <option value='difficult'>Difficult</option>
</select>

Textareas

Note that interpolation on textareas will not work:

<textarea>{{ textContent }}</textarea>

Use v-model instead:

<textarea v-model='textContent'></textarea>

Modifiers

The v-model directive supports 3 modifiers:

  • .number: Will cast the string input to a numerical value
  • .trim: Will trim any white space at the beginning and end of the string input
  • .lazy: Will only update the associated data property once the field is inactive
