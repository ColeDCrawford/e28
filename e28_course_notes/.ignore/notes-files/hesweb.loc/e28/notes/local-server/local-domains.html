Local domains

Following the local server notes up to this point, you should have created an application called practice that exists as a copy on Github.com and also your computer.

Up until this point, we accessed our local server by visiting http://localhost which loaded the contents of our XAMPP htdocs folder since that’s what the server’s document root was configured to.

We could access the practice application via http://localhost/e28/practice but that serves the application as a subdirectory, which we don’t want. Instead, we want to run the application as we would on a production server, via its own domain or subdomain.

To do this, we could change the document root via Apache’s config file so that it pointed directly to the htdocs/practice directory. With this change, visiting http://localhost would load our practice application.

However, when we start working with multiple applications, it can be tedious to have to reconfigure the document root every time we switch which application we’re working on.

A better solution is to create unique local URLs that are configured for each application we build, with their own custom document root.

For example, we’ll start by configuring a URL http://practice.loc that points to htdocs/e28/practice as its document root.

Following this example, you’ll do the same thing for each of the 3 unique applications you build for the projects in this course:

  • http://e28p1.loc => htdocs/e28/p1
  • http://e28p2.loc => htdocs/e28/p2
  • http://e28p3.loc => htdocs/e28/p3

In order to do this, we’ll configure our Apache web servers to run multiple applications via Virtual Hosts.

Step 1. Tell Apache to use the virtual hosts file

First, locate and open httpd.conf, your local server’s Apache configuration file:

  • Mac: /Applications/XAMPP/xamppfiles/etc/httpd.conf
  • Windows: c:\xampp\apache\conf\httpd.conf

Edit this file so that the line including httpd-vhosts.conf is not commented out by removing the pound sign from the start of the Include:

Mac:

# Virtual hosts
Include etc/extra/httpd-vhosts.conf

Windows:

# Virtual hosts
Include conf/extra/httpd-vhosts.conf

Step 2. VirtualHost entry

Next, tell Apache on your local server how to handle requests to http://practice.loc via a VirtualHost entry.

This is done via Apache’s httpd-vhosts.conf config file which can be found at the following locations:

  • Mac: /Applications/XAMPP/etc/extra/httpd-vhosts.conf
  • Windows: c:\xampp\apache\conf\extra\httpd-vhosts.conf

Within this file, add your own VirtualHost block:

Mac users, yours will look like this:

<VirtualHost *:80>
    ServerName practice.loc
    DocumentRoot /Applications/XAMPP/xamppfiles/htdocs/e28/practice
    <Directory /Applications/XAMPP/xamppfiles/htdocs/e28/practice>
        AllowOverride All
        Options -Indexes
        Require all granted
    </Directory>
</VirtualHost>

Windows users, yours will look like this:

<VirtualHost *:80>
    ServerName practice.loc
    DocumentRoot c:\xampp\htdocs\e28\practice
    <Directory c:\xampp\htdocs\e28\practice>
        AllowOverride All
        Options -Indexes
        Require all granted
    </Directory>
</VirtualHost>
FYI

The above VirtualHost blocks assumes you’re running Apache on Port 80 (<VirtualHost *:80>). If you’re using a different port, make that edit and note that you would then have to access your local domain via a URL such as http://practice.loc:8888 (where 8888 is whatever alternative port you’re using)

After you’ve added a VirtualHost block, restart Apache to make the changes take effect.

If Apache won’t restart, it’s possible you have a bug/syntax error in your VirtualHost blocks; see the troubleshooting section below.

FYI

Optional: If you want to learn more about each line in the above provided VirtualHost template, read these supplemental notes on Apache configs.

Step 3. Create a new host

Next, we need to edit your computer’s hosts file which can be used to route domains to an IP address of your choice.

  • Mac: /private/etc/hosts
  • Windows: c:\Windows\System32\drivers\etc\hosts

(Note, there’s no extension on this file, it’s simply called hosts)

The hosts file is protected, so you’ll need to open it with administrator privileges. This can be done with nano via the admin command sudo on Mac, or elevate on Windows.

Mac:

$ sudo nano /private/etc/hosts

Windows:

$ elevate nano c:\Windows\System32\drivers\etc\hosts

At the bottom of your hosts file, add a new host:

127.0.0.1 practice.loc

This is telling your computer that whenever you access http://practice.loc from your computer, it should map to the ip address 127.0.0.1 (the IP address of your local server).

Test it

With your local domain configured in both your server’s VirtualHost file and your computer’s hosts file, you’re ready to test it out by visiting http://practice.loc in your browser.

Make sure you explicitly type in http://practice.loc with http:// at the beginning. If you don’t, your browser may try and do a web search for practice.loc because it does not recognize .loc as a domain extension.

Fixing http://localhost

After you make the above change you’ll notice that http://localhost no longer works as it did before— instead of pointing to XAMPP’s /htdocs directory, it will start pointing to the first VirtualHost block in your httpd-vhosts.conf file.

To fix this, you can add a VirtualHost block specifically for http://localhost. I recommend having localhost just point to your htdocs folder.

Mac:

<VirtualHost *:80>
    ServerName localhost
    DocumentRoot /Applications/XAMPP/xamppfiles/htdocs
    <Directory /Applications/XAMPP/xamppfiles/htdocs>
        AllowOverride All
        Options -Indexes
        Require all granted
    </Directory>
</VirtualHost>

Windows:

<VirtualHost *:80>
    ServerName localhost
    DocumentRoot c:\xampp\htdocs
    <Directory c:\xampp\htdocs>
        AllowOverride All
        Options -Indexes
        Require all granted
    </Directory>
</VirtualHost>

Don’t forget to restart your server after making these changes.

Summary

To be repeated every time you want to add a new app:

  • Add a new <VirtualHost> record block in Apache’s httpd-vhosts.conf file.
  • Add a new local URL in your computer's host file.

Note how it’s only Steps 2 and 3 above that need to be repeated for any new apps. Step 1 is a one time deal to get Virtual Hosts working.

Troubleshooting

Apache won’t start after editing VirtualHosts file (httpd-vhosts.conf)

Small syntax errors in the VirtualHosts file can cause Apache to fail to load, and such parse-related errors won’t show up in your Apache error log file.

To debug these issues, invoke Apache directly from the command line using the -S flag (ref).

Mac:

$ /Applications/XAMPP/bin/httpd -S

Windows:

$ c:\xampp\apache\bin\httpd.exe -S

This will show you the parsed settings from your VirtualHost file and if there are errors it will tell you on which lines.

Here’s an example of this technique, shown with the results of a missing space in one of my VirtualHosts blocks:

$ /Applications/XAMPP/bin/httpd -S
AH00112: Warning: DocumentRoot [/Applications/XAMPP/xamppfiles/docs/dummy-host.example.com] does not exist
AH00112: Warning: DocumentRoot [/Applications/XAMPP/xamppfiles/docs/dummy-host2.example.com] does not exist
AH00526: Syntax error on line 46 of /Applications/XAMPP/xamppfiles/etc/extra/httpd-vhosts.conf:
Invalid command 'AllowOverrideAll', perhaps misspelled or defined by a module not included in the server configuration
