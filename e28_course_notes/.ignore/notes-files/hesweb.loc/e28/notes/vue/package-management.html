Package management

Package management software keeps track of all the dependencies that a project utilizes.

Done via a config file in the root of the project. This config file lists the dependencies and their version numbers. Example:

"dependencies": {
    "vue": "2.6.11",
    "bootstrap": "^4.1.0"
    // etc...
},

Version numbers

Version numbers are always given in the format of major.minor.patch.

In the above example, we specify vue version 2.6.11 so that’s major version 2, minor version 6, patch version 11.

The bootstrap version number includes a special character, ^ which introduces some flexibility by indicating we want the most up to date version for the major version number given (4). So even though we specified 4.1.0, we’d actually get the most recent 4.x.x version which is (as of this writing) 4.5.3.

By using flexible version numbers like this, our package manager can help ensure we’re keeping our software up to date with the most recent minor/patch releases.

Lock files

Most package managers will create a lock file which writes the specific version numbers of the software it downloaded. This lock file is then used to load packages on production servers, ensuring that our local/development servers are kept perfectly in sync with our production servers.

Package managers in other contexts

Many languages have package managers - either official or 3rd party ones. E.g.:

  • Python: Pip, PyPI, etc.
  • Java: Gradle, Maven, etc.
  • Ruby: RubyGems
  • PHP: Composer
  • JavaScript: NPM, Yarn, etc.

Package managers for global packages

In addition to managing the dependencies of a specific project, package managers can also be used to download packages/programs that can be installed globally on your computer.

As an example, we’ll be using NPM to download the command line program Vue CLI, which we’ll use to generate and configure Vue projects on our computers.

Given that, our first step in this whole process is to get NPM installed...

NPM (Node Package Manager)

  • Default package manager for Node.js (the server-side implementation of JavaScript), but used in the client-side world as well
  • Command line tool
  • Works with the NPM registry - an online database of public and paid-for packages
  • Config file: package.json
  • Packages are installed in a node_modules directory

Install NPM

See if you have NPM installed by running the command npm

$ npm

Usage: npm <command>

where <command> is one of:
    access, adduser, audit, bin, bugs, c, cache, ci, cit,
    clean-install, clean-install-test, completion, config,
    create, ddp, dedupe, deprecate, dist-tag, docs, doctor,
    edit, explore, get, help, help-search, hook, i, init,
    install, install-ci-test, install-test, it, link, list, ln,
    login, logout, ls, org, outdated, owner, pack, ping, prefix,
    profile, prune, publish, rb, rebuild, repo, restart, root,
    run, run-script, s, se, search, set, shrinkwrap, star,
    stars, start, stop, t, team, test, token, tst, un,
    uninstall, unpublish, unstar, up, update, v, version, view,

npm <command> -h  quick help on <command>
npm -l            display full usage info
npm help <term>   search for help on <term>
npm help npm      involved overview

Specify configs in the ini-formatted file:
    /Users/Susan/.npmrc
or on the command line via: npm <command> --key value
Config info can be viewed via: npm help config

If you don’t have it installed, you can download and installer from https://nodejs.org/en/download.

(Note: to get NPM you’re actually installing Node.js which comes bundled with NPM)

In order to run the npm from command line, the path to the npm executable must be in your computer’s PATHs file. This file specifies which paths to look in when executing command line programs.

Windows users - The installer automatically added NPM executable path to your computer’s PATH settings, so you don't have to do anything else. Close and restart your command line program and you should have access to the npm command.

Mac users - You will need to manually update your computer’s PATHs file.

Open the file as an admin using nano:

sudo nano /etc/paths

Add the following lines to this file:

/usr/local/bin/node
/usr/local/bin/npm

Save your changes and exit nano.

Restart Terminal and you should now be able to execute the npm command.
